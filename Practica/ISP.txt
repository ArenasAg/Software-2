4. Principio de Segregación de Interfaces (ISP - Interface Segregation Principle)
Descripción:
Los clientes no deberían estar obligados a depender de interfaces que no utilizan. 
Las interfaces deben ser específicas para cada tipo de cliente.


// INCORRECTO: Todos los tipos de trabajadores deben implementar métodos que no usan
public interface Worker {
    void work();
    void eat();
}

public class Robot implements Worker {
    @Override
    public void work() {
        // Lógica de trabajo
    }

    @Override
    public void eat() {
        // No es relevante para un robot
        throw new UnsupportedOperationException();
    }
}

// CORRECTO: Separar la lógica en diferentes interfaces
public interface Workable {
    void work();
}

public interface Eatable {
    void eat();
}

public class Robot implements Workable {
    @Override
    public void work() {
        // Lógica de trabajo
    }
}

public class Human implements Workable, Eatable {
    @Override
    public void work() {
        // Lógica de trabajo
    }

    @Override
    public void eat() {
        // Lógica de comer
    }
}
